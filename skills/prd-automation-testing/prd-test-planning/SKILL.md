# PRD Test Planning

## 目标

将模块 `PRD` 或产品原型图转换为可执行的自动化自测规划资料。此 skill 只负责生成和维护 `01-04`，不进入页面执行、字段采集、失败取证或视频录制。

## 输入门禁

至少需要 `PRD` 或产品原型图二选一。两者同时存在时，以 `PRD` 为准，原型图只作为辅助解释。

规划阶段允许缺少测试路径、登录方式、测试账号、权限说明和测试数据，但必须在输出资料中标记缺失项，并在生成 `01-04` 后主动向用户确认。

如果缺少模块名称，不创建模块目录；先询问用户补齐模块名称。

## 工作流

1. 读取 `PRD` 或原型图，提取模块边界、功能点、字段基线、操作场景、权限要求和风险动作。
2. 在 `docs/testing/<模块中文目录>/` 下生成规划资料：
   - `01-模块拆解.md`
   - `02-测试用例.md`
   - `03-执行清单.md`
   - `04-测试数据与账号.md`
3. 所有功能点和用例必须带 `F-xxx` 追踪标识，测试用例必须能回连到 `01-模块拆解.md`。
4. 用例必须按 `P0/P1/P2/P3` 分级，并预分配执行目录 `执行结果/<优先级>/`。
5. 生成 `01-04` 后暂停，等待用户确认，不自动进入执行。

## 确认顺序

生成规划资料后，按以下顺序补齐执行前门禁：

1. 让用户确认测试用例、字段基线、优先级、账号权限、测试数据和高风险动作授权。
2. 如果没有测试路径，主动询问菜单路径或 URL，不得猜测入口。
3. 用户提供测试路径后，主动询问登录方式：指定测试账号登录、复用当前已登录状态，或无需登录直接访问。
4. 测试路径和登录方式都确认后，再让用户决定执行模式：单 agent 独立执行，或多 agent 并行执行。

未完成上述确认时，只允许更新规划资料，不允许启动浏览器自动化验证。

## 字段基线规则

默认提取并记录以下区域：

1. 查询区字段。
2. 列表表头字段。
3. 详情展示字段。
4. 关键操作按钮。
5. 关键弹窗字段。

字段来源必须明确标注为 `PRD`、原型图或用户补充口径。只有原型图作为来源时，必须在文档中标记“原型图来源，待用户确认”。

## 模板

优先复用本 skill 的模板：

- `templates/01-模块拆解.md`
- `templates/02-测试用例.md`
- `templates/03-执行清单.md`
- `templates/04-测试数据与账号.md`
