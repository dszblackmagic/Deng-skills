# 执行清单

## 1. 资料确认门禁

用户直接提供 `PRD` 或产品原型图时，第一轮只生成 `01-04` 规划资料。进入页面执行前，必须完成以下确认：

| 确认项 | 通过标准 | 记录 |
| --- | --- | --- |
| 测试用例确认 | 用户确认 `02-测试用例.md` 可执行 |  |
| 字段基线确认 | 用户确认字段来源与预期字段清单 |  |
| 优先级确认 | 用户确认 `P0/P1/P2/P3` 分级合理 |  |
| 测试路径确认 | 用户确认菜单路径或 URL；缺失时必须先询问 |  |
| 登录方式确认 | 用户确认指定账号登录、复用当前登录态或无需登录 |  |
| 账号权限确认 | 用户确认账号、权限、测试数据可用 |  |
| 高风险授权确认 | 用户确认是否允许新增、编辑、终止、导出等动作 |  |
| 执行模式确认 | 用户选择单 agent 独立执行或多 agent 并行执行 |  |

未完成上述确认时，不进入分级执行阶段。

## 2. 执行前检查

| 检查项 | 操作 | 通过标准 | 记录 |
| --- | --- | --- | --- |
| 测试路径 | 确认菜单路径或 URL | 已由用户提供并确认 |  |
| 登录方式 | 确认指定账号登录、复用当前登录态或无需登录 | 已由用户确认 |  |
| 环境可访问 | 打开测试环境 | 页面正常加载 |  |
| 会话状态 | 判断是否已登录 | 已登录则复用；未登录则登录 |  |

## 3. 执行模式

| 执行模式 | 执行规则 | 写入范围 |
| --- | --- | --- |
| 单 agent 独立执行 | 按 `P0 -> P1 -> P2 -> P3` 顺序执行 | 仍然写入 `执行结果/<优先级>/` |
| 多 agent 并行执行 | 按 `P0/P1/P2/P3` 分派 agent | 每个 agent 只写所属优先级目录 |

说明：无论采用哪种执行模式，根目录 `05-执行记录.md` 和 `06-缺陷记录.md` 都只由主流程在汇总阶段生成或更新。

## 5. 分级执行批次

测试用例执行时按优先级拆分到独立目录：

| 优先级 | 执行目录 | 执行范围 | 记录要求 |
| --- | --- | --- | --- |
| P0 | `执行结果/P0/` | 页面进入、核心入口、主链路 | 写入该目录下的 `05-执行记录.md` 和 `06-缺陷记录.md` |
| P1 | `执行结果/P1/` | 核心验收、关键字段、主要查询和详情 | 写入该目录下的 `05-执行记录.md` 和 `06-缺陷记录.md` |
| P2 | `执行结果/P2/` | 风险、异常、权限、边界 | 写入该目录下的 `05-执行记录.md` 和 `06-缺陷记录.md` |
| P3 | `执行结果/P3/` | 低频、观察性或补充回归 | 写入该目录下的 `05-执行记录.md` 和 `06-缺陷记录.md` |

## 6. 失败分流

| 失败类型 | 处理方式 |
| --- | --- |
| 字段类失败 | 截图 + 差异说明 |
| 操作类失败 | 截图 + Console + Network + `playwright-cli` 录视频 |

说明：通过用例不需要截图证明，只在执行记录中填写实际结果摘要。截图只用于字段差异、失败、阻塞、高风险提示或其他需要保留现场的场景。

说明：若因账号、权限、测试数据、环境异常等原因无法继续执行，统一记为“阻塞”，先记录到所属优先级目录的 `05-执行记录.md`，默认不直接登记为缺陷。

## 7. 汇总规则

各优先级目录执行完成后，再将结果汇总到模块根目录：

| 汇总文件 | 来源 | 汇总内容 |
| --- | --- | --- |
| `05-执行记录.md` | `执行结果/P0/` 至 `执行结果/P3/` | 执行批次、通过、失败、阻塞、未测、证据索引 |
| `06-缺陷记录.md` | `执行结果/P0/` 至 `执行结果/P3/` | 已确认缺陷、字段差异缺陷、操作类缺陷、证据索引 |
