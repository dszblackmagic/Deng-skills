# 执行记录

说明：本模板可用于优先级目录下的独立执行记录，也可用于模块根目录的最终汇总执行记录。优先级目录中的记录只写本级别用例；根目录中的记录只写汇总结果和证据索引。

## 状态说明

- `通过`：执行结果与 `PRD` 预期一致。
- `失败`：已完成执行且确认存在偏差，可继续判断是否进入缺陷。
- `阻塞`：因账号、权限、测试数据、环境或依赖条件问题无法继续执行，先记录阻塞原因，默认不直接进入缺陷。
- `未测`：本轮未执行或被排期跳过。

证据规则：通过用例不需要截图证明，截图证据和视频证据可留空；字段差异、失败、阻塞或高风险提示场景按规则补充证据。
